The Elevator

  • The elevator that I used for this project is located at the court house in Big Lake, Tx.

    Gif of the elevator in use

  • The issue with this current design is that the visually impaired would not be able to operate it. There is no Braille anywhere on the elevator control panel. Also, another issue could be that there is no audio feeback when the door is opening or closing. There is only audio feedback when you're selecting your desired floor.
  • I believe that this elevator design is decent. There is a visual feedback when operating the control panel. The buttons are placed neatly as in where anyone is able to access them. There is also an emergency light located on the top of the control panel.

What About This Elevator?

  • The most common times I use an elevator is when I get my apartment. I also used an elevator many times to take heavy objects up floors. I believe this interface could make doing these common things easier because the design is simple and easy to understand.
  • The most common sequence of actions when a user interacts with the elevator goes like this:
    1. Call the elevator by pressing the up or down button.
    2. Wait for the elevator to arrive.
    3. Once inside, press the button to the desired floor.
    4. Wait till the elevator arrives at the desired floor.
    5. Exit the elevator.
  • The way this elevator supports the user figure out how to make it work is by including pictures by each button. Also, there is this black label surrounding the button and the corresponding picture.
  • The feedback that is provided is audio and visual. There is a visual feedback when pressing a button (the red ring lighting up around the button) and an audio feedback when pressing a button (a loud beep).
  • A common mistake with this design is that the keyhole located close to the buttons could be mistaken as a button itself. The reason this being is because it is surounded by the black label like the other buttons. Another common mistake in this elevator could be that you might not see the call elevator button because it is the same color as the back panel.
  • Some improvements I would make on this elevator is that it needs to be visually impaired accessible. To help with this I would add Braille onto the label of the buttons. Another improvement I would make is to change the color of the buttons. I think that changing the color of the buttons to not match the background might make it easier to see where every button is. My last improvement would be to add audio feedback when the doors are opening and closing. With this, people will watch for the elevator doors just in case they might not be looking or paying attention.
